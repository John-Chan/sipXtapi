About extension pools

Work with extension pools

For each SIPxchange user, you supply a unique, alphanumeric User ID. If your User IDs contains letters or other characters, you can also set up a unique, numeric extension for each user to streamline call addressing and allow phone top voicemail access.

  • If you assign numeric-only User IDs to your users, separate extensions are not needed. Extension pools are not used.
     

  • If you assign User IDs that include characters, each user also needs an extension. You can use extension pools to manage the set of usable extensions and keep track of numbers that have already been assigned to users.

When you add a new user, you can supply an extension either by typing in an extension number, or by choosing the next available extension from a pool. Since SIPxchange extensions must be unique to each user, using an extension pool can help speed up the assignment and data entry process.

Extension pools are not hierarchical, and each extension number can only be assigned to a single pool. You might set up just one pool, or several pools to reflect different geographical locations, departments, or other organizational structures.

When you install SIPxchange, one extension pool is created automatically. For more information about this special extension pool, see About the reserved extension pool.
