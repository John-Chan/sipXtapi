About extension pools

Work with extension pools

When you install SIPxchange, one extension pool is created automatically. The purpose of this extension pool, named "reserved," is to assure that certain extensions, which you may have set up in some system outside of the Configuration Server, are not used to identify users.

For example, the Media Server's Auto Attendant and Voicemail applications each have an identifying numeric address, which you configure in the list of system-wide dial plans. If you assign addresses of 100 and 101, respectively, to the Auto Attendant and Voicemail, these values should not be inadvertently assigned to some other user.

To assure that certain numeric identifiers, such as the Auto Attendant and Voicemail extensions, remain unique, you add those extensions to the reserved extension pool. Each time you create a user, the values you supply for the User ID and the extension are compared to the extensions in the reserved pool.

Tip: Even if your User IDs are numeric only, you may wish to add one or more extensions to the reserved extension pool.
