About devices

Work with devices

To help you manage a changing inventory of SIP phones effectively, the Configuration Server makes a distinction between an actual SIP user agent, such as a Pingtel or Cisco phone or a PC softphone application, and the logical use to which a phone will be put in an organization, such as its location (main lobby, residence hall, loading dock) or the job title of its regular user (customer support representative, CEO, sales).

With the Configuration Server, you set up a “device” to represent each logical phone entity in your organization.

As a result, rather than deliver configuration settings and software to a specific physical phone based only on its MAC address, serial number, or other identifier, the Configuration Server determines what configuration settings and software to deliver to a SIP phone based on its logical (device) deployment.  When you replace a malfunctioning phone appliance with a new, physically different phone, the Configuration Server can continue to deliver the correct configuration settings and software to the new phone.

You assign zero, one, or more devices to each SIPxchange user.
