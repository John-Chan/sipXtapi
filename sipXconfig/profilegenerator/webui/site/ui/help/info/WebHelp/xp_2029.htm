Additional user lines

A user line provides an address for a user or group, which applies in turn to all the phones associated with devices assigned to that person or group.  User lines help assure that:

  • Incoming calls can be routed to the correct phone.

  • Outgoing calls are identified with their source.

When you set up a user, the Configuration Server automatically creates a primary user line.

You can add more user lines as needed: click the add icon  .  Each user line consists of:

  • A SIP URL, which identifies the user to your network, and is included in SIP messages sent by the phone.

  • Whether or not the phone should periodically send registration messages for this line to another network resource, such as a SIP Registry server.

  • Whether or not the user's call forwarding settings should apply to the incoming calls addressed specifically to this line.

When you click Submit, the line's values are stored together in a numbered set.

You can edit or delete any of the user lines that you set up: click the edit icon    or the delete icon  .

You can also define sets of authentication credentials for any additional lines you add.

Parameter names

USER_LINE.<n>.URL
USER_LINE.<n>.REGISTRATION
USER_LINE.<n>.ALLOW_FORWARDING

Type

URL stores a string; REGISTRATION and ALLOW_FORWARDING are enumerations

Values

for REGISTRATION, choose REGISTER or PROVISION
for ALLOW_FORWARDING, choose ENABLE or DISABLE

Examples

You set up a user, John Doe. In addition to the primary line set up by the Configuration Server, you set up a second user line so that John can receive calls addressed to his group, the support group:
USER_LINE.2.URL : sip:support@regsrvr.myco.com
USER_LINE.2.REGISTRATION : REGISTER
USER_LINE.2.ALLOW_FORWARDING : DISABLE

User Category

Lines

The Configuration Server also sets up device lines automatically, one for each device you create.

When you assign a device to a user, all of the user's lines are associated with the device so that calls can be received and made.

SIP URL

SIP URLs are similar to email addresses: they require a resource identifier and a host in this format: sip:resourceID@host. The resource ID you assign to a user line will generally identify a person or group.

In addition to the required elements, you can include optional elements:
"displayname"<sip:resourceID@host:port;URLparam=n>;fldparam=n

You can reuse the same SIP URL in lines that you set up for different users. For example, you can set up a user line with a SIP URL of sip:support@regsrvr.myco.com for each member of the support group.

Registration

For each line, you specify whether or not the associated phone(s) should send registration messages to the host in that line’s SIP URL.

  • For installations that include a SIP Registry server, you are likely to want the line to be registered. For the user's associated phone(s) to send registration messages to a SIP Registry server or other network resource automatically, you must choose to register the line and include the server’s fully qualified domain name as the host in the line’s SIP URL.

  • For installations that do not have a SIP Registry server, or if you do not want the phone to register a line, you indicate that the line is provisioned as is.

Call forwarding

You (or the end user) set call forwarding properties for each user. These properties include whether or not to forward incoming calls in different situations (when busy, on no answer, or all calls), and the destination for each case. When you set up a user line, you specify whether or not the defined call forwarding preferences should apply to that line.

  • If you enable call forwarding for a line, the user's call forwarding settings apply to all incoming calls addressed to that line.

  • If you disable call forwarding for the line, incoming calls made to that line are not forwarded.

Reminder: This setting does not apply to SIPxchange voicemail. Regardless of the value you supply for this setting, calls will be redirected to voicemail for users who have the voicemail inbox permission. If you enable the Apply rules for call forwarding setting for a line, it applies only when the user has call handling settings defined to override SIPxchange voicemail.

Authentication credentials

When a phone receives a challenge for authentication from another SIP user agent or server, it responds by sending the user name and password required by the realm that is making the challenge. To prepare a phone so that it can respond to authentication challenges with valid data, you supply the realm, user name, and password of each SIP user agent and server that requires authentication. These values are case sensitive.

You can enter any number of credential sets for a line:

  1. Click the add icon    next to Credentials for the line.

  2. Enter a user ID.

  3. Enter the realm.

  4. Enter and confirm the password.

  5. Click Submit.

The user ID, realm, and password values are stored together in a numbered set. The credential sets you define for each line are independent: you must store the same credentials for every line, if necessary.

To edit or delete a credential set: click the edit icon    or the delete icon  .

Properties

USER_LINE.<n>.CREDENTIAL.<n>.REALM
USER_LINE.<n>.CREDENTIAL.<n>.USERID
USER_LINE.<n>.CREDENTIAL.<n>.PASSTOKEN

Type

String

Examples

To respond to challenges from the "Earthsea" realm, you store the valid user name "guest" and its password "welcome". The password is encrypted using the Digest Access Authentication Scheme.
USER_LINE.2.CREDENTIAL.9.REALM : Earthsea
USER_LINE.2.CREDENTIAL.9.USERID : guest
USER_LINE.2.CREDENTIAL.9.PASSTOKEN : e7cf19(etc.)
