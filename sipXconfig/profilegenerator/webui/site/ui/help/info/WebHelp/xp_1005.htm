Authorized phone users

This setting establishes user names and passwords for access to a Pingtel phone's embedded web server. You must also enable HTTP or HTTPS access to the web server with the HTTP port or Secure HTTPS port setting.

One authorized phone user name, admin, is supplied for each device with a default password of null. System administrators should be sure to provision each phone with a non-null admin password.

The admin user name and its password authorize access to:

  • Certain functions you perform on the phone itself, including network settings.

  • Telnet access (if activated).

  • The phone's browser-based interface, which in a SIPxchange system is used only for troubleshooting and other diagnostic procedures.

Additional user IDs and passwords allow access to a subset of the phone's browser-based interface and the MyPingtel user portal only, which generally are not used in SIPxchange systems.

To add an authorized user:

  1. Click the add icon  .  A popup window opens.

  2. Enter an authorized user name.

  3. Enter the password.

  4. Re-enter the password to confirm.

  5. Click Submit.

After you complete all of your edits for the device or device group, click Save.
