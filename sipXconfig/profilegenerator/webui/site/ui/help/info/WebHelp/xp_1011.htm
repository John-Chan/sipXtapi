Starting port for RTP/RTCP packets

Pingtel phones use a consecutively numbered pair of ports to receive audio media and control information for each concurrent connection. RTP (Real-time Transport Protocol) packets carry audio media, while RTCP (Real-time Control Protocol) packets carry control information. With this setting, you define the starting port in the RTP/RTCP pair of ports.

Because a Pingtel phone can maintain multiple concurrent connections, several consecutively numbered ports may be identified when you set this parameter. For example, to maintain five connections, this setting indicates the first of ten ports:
(1 RTP port + 1 RTCP port) x 5 connections = 10 ports.

This setting also applies to devices that make calls through a packet-based firewall.

Parameter name

PHONESET_RTP_PORT_START

Type

Integer

Default if blank

8766

Device Category

Firewall
