Manage Gateways

You identify the IP-to-PSTN gateway(s) that SIPxchange should use for routing outbound calls on this page. You can specify different gateways to use for routine outbound calls and to use for emergency (911) calls.

Generally, you must set up at least one gateway for routine calls. If you do not set up a separate emergency gateway, the routine gateway is used to route both routine and emergency calls.

If you define more than one gateway for either routine or emergency calls, SIPxchange tries each one in sequence: the first gateway is tried first. If that gateway rejects the call, the next gateway is tried, then the next, etc.

Tip: If your SIPxchange installation has complicated call routing needs, you may not be able to use this procedure to configure your gateways. Please contact Pingtel’s TAC for more information.

Specify gateways

To specify the address of an IP-to-PSTN gateway for your SIPxchange system:

  1. Click Configuration in the left column.
  2. Find Gateways on the list of services.
  3. Click the icon in the Configure column for Gateways. A new page lists currently defined gateways.
  4. You can add

    • Regular IP-to-PSTN gateways for routine calls

    • Emergency IP-to-PSTN gateways

  5. You add gateways by clicking   in the appropriate category.

  6. For each gateway, you supply a primary address. The address can be an IP address or fully qualified host name. To specify an optional port, use <address>:<port> format.

  7. To assure that SIPxchange properly authenticates all calls routed to an IP-to-PSTN gateway, you should provide all other system addresses, including alternate DNS names, that can be used to reach the gateway. Add an alternate address by clicking    next to Alternates.
  8. If you define more than one gateway in a category, you can change gateway prioritization by clicking Move Up or Move Down.
  9. You make edits to an existing gateway by changing the text in the text boxes.
  10. Click Save when you complete your edits.

Your new settings do not take effect until you restart.
